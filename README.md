# DevBlog(React)
React를 사용하여 제작한 개발자 블로그 애플리케이션입니다.

---

## 1. 프로젝트 개요

### 1.1. React 핵심 개념
- **SPA** <br/>
React Router을 활용하여 페이지 전환 시 전체 새로고침 없이 컴포넌트만 교체합니다. `<Routes>`를 선언하여 경로를 지정하고, 인증 여부에 따라 `<PrivateRoute />`와 `<PublicOnlyRoute>`로 접근을 제어합니다.

- **컴포넌트 기반 설계** <br/>
UI는 `components/`폴더 하위에 각각 `layout`, `section`, `common`, `comment`, `post`로 구분하고, 비즈니스 로직은 `hooks/`로 분리했습니다. API 통신은 `api/` 폴더에 모아두어 뷰 컴포넌트가 HTTP 구현에 의존하지 않도록 설계했습니다.

- **REST API** <br/>
Axios 인스턴스를 중앙에서 관리하며, 요청 인터셉터로 JWT 토큰을 자동 첨부하고 응답 인터셉터로 401 만료 시 자동 로그아웃 처리를 구현했습니다.

- **URL 기반 상태 관리** <br/>
`usePostQuery` 사용자 훅으로 URL 검색 파라미터를 상태로 사용하여, 뒤로가기/공유/새로고침에도 필터/정렬/검색 상태가 유지되도록 설계했습니다.

### 1.2. React를 선택한 이유

- 컴포넌트 단위 재사용으로 UI 일관성을 유지하기 편합니다.
- Virtual DOM 기반 렌더링으로 목록 업데이트가 많은 블로그에 적합합니다.
- 풍부한 에코시스템으로 빠른 개발이 가능합니다.

## 3. 핵심 기능

### 3.1. 게시글 CRUD
- **작성**: Markdown 에디터로 내용을 작성하고, 태그를 Enter/Blur로 추가합니다. 썸네일의 경우 Cloudinary에 업로드하고 URL을 DB에 저장하는 방식입니다.
- **조회**: 게시글 상세 페이지 진입 시 조회수(PATCH /posts/:id/view)를 자동으로 증가시킵니다.
- **수정/삭제**: 작성자와 현재 로그인한 유저를 대조하여 작성자가 일치할 경우에만 사용 가능하며, 서버와 클라이언트 모두에서 권한을 검증합니다.

### 3.2. 무한 스크롤
- `IntersectionObserver`로 목록 마지막 요소를 감지하여 다음 페이지를 자동으로 로드합니다.
- `usePosts` 훅이 파라미터를 받아 데이터를 누적 관리하며, 조건 변경 시 페이지를 1로 초기화하고 목록을 새로 불러옵니다.
- 데이터 가공은 서버에서 수행합니다.

### 3.3. 게시글 필터링 및 검색
- 게시글에 해당하는 태그 목록을 클릭하면 URL의 `tag` 파라미터가 업데이트되어 해당 태그 게시글만 표시합니다.
- 헤더의 검색창은 게시글 목록 페이지에서는 실시간으로, 다른 페이지에서는 Enter로 `/posts?query=검색어`로 이동합니다.
- 정렬은 최신순 / 조회순 / 좋아요순(예정) 3가지를 지원합니다.

### 3.4. 밝기 테마
- `ThemeContext`에서 CSS 커스텀 속성으로 테마를 전환합니다.
- 최초 진입 시 미디어 쿼리로 시스템 테마를 감지하고, 로컬스토리지에 저장하여 재방문 시 유지합니다.

### 3.5. JWT를 활용한 회원 인증
- 회원가입 시 bcryptjs로 비밀번호를 해싱하여 저장합니다.
- 로그인 성공 시 서버에서 JWT를 발급하고, 클라이언트는 Zustand persist로 토큰을 로컬스토리지에 유지합니다.
- 모든 보호 라우트는 `authMiddleware`가 헤더를 검증합니다.
- Axios 응답 인터셉터가 401 응답 감지 시 자동으로 로그아웃 후 `/login`으로 리다이렉트합니다.

### 3.6. 댓글
- 로그인한 사용자만 댓글을 작성할 수 있으며, 본인 댓글만 삭제 버튼이 노출됩니다.
- 댓글 생성/삭제 시 Post의 `commentCount` 필드를 업데이트합니다.

### 3.7. 프로필 설정
- 닉네임, 이메일, 자기소개, 프로필 이미지, 비밀번호를 수정할 수 있습니다.
- 변경된 필드만 골라 `PATCH` 요청을 보내고, 비밀번호 변경 시 현재 비밀번호를 추가로 요구해 검증합니다.
- 프로필 이미지는 Cloudinary에 업로드 전 `URL.createObjectURL`로 미리보기를 제공하며, 설정된 프로필이 없을 경우 기본 이미지로 표시됩니다.

## 4. 트러블 슈팅

### 4.1. 반응형 사이드바 오픈 시 스크롤바 사라짐과 동시에 레이아웃 흔들리는 문제
**문제**: 모바일 크기에서 사이드바 열림 시 `overflow: hidden` 적용으로 스크롤바가 사라지면서 콘텐츠가 밀려 UX를 해치는 문제
**원인**: 스크롤바 너비만큼 레이아웃이 이동하는 현상입니다.
**해결**: `useLockBodyScroll`에서 스크롤바 너비 계산 로직을 추가하고 `body`에 해당 값만큼 패딩을 추가했습니다. 

### 4.2. Axios 인터셉터에서 로그인/회원가입 요청도 로그아웃되는 문제
**문제**: 로그인 실패(401) 시에도 응답 인터셉터가 로그아웃을 실행하여 항상 `/login`로 리디렉션 되는 문제
**원인**: 인터셉터가 모든 401 응답을 동일하게 처리했습니다.
**해결**: `public__paths` 배열에 제외할 경로값을 정의하고, 해당 경로에서 발생한 401은 로그아웃을 건너뛰도록 예외 처리했습니다.

### 4.3. 무한 스크롤 데이터 중복 문제
**문제**: `req.query.page`값이 항상 1로되어 서버가 같은 값을 계속 보내는 문제
**원인**: `axios.get`은 두 번째 인자로 `config` 객체를 받는데, 데이터를 단순한 객체 형태로 전달하여 Axios가 이를 유효한 설정으로 인식하지 않아 문제가 발생했습니다.
**해결**: `params`키를 사용하여 데이터를 전달함으로써, Axios가 이를 URL 쿼리 스트링으로 변환하여 요청하도록 수정했습니다.

## 5. 개선 사항

### 5.1. 활동 / 통계 페이지 구현
사이드바에 활동과 통계 메뉴가 있으나 실제 페이지는 미구현 단계입니다. 내가 쓴 글, 댓글 이력, 조회수 및 좋아요 추이 등을 시각화하는 대시보드 추가를 고려하고 있습니다.

### 5.2. 좋아요 기능 완성
현재 좋아요 API는 구현되어 있으나, 프론트엔드 UI에 연동이 안되었고 좋아요 기반 필터링도 미구현 상태입니다.

### 5.3. 에러 처리 고도화
현재 에러 발생 시 `<div>에러 발생</div>`만 표시됩니다. 에러 유형에 따라 사용자 친화적인 에러 페이지나 Toast 알림 제공할 필요가 있습니다.

### 5.4. UX 개선
브레드크럼과 검색 미리보기 창, 로딩 시 스켈레톤 이미지 구현을 준비 중입니다.

### 5.5. SEO 최적화
`react-helmet-async`를 활용하여 검색 노출을 최적화할 예정입니다.

### 5.6. 유저 상세 페이지
다른 사람의 게시글 및 댓글 목록을 확인하 수 있는 유저 상세 페이지 구현을 고려하고 있습니다.
